## 0. 設計立場（先講清楚）

本專案**不是「內容空洞」**，而是：

1. **系統層已經偏厚**（元素克制、事件波、詞綴、英雄抽卡、女神、成就、神魔誌、顧問）。
2. **局中體感仍偏「同一條無盡線」**：地圖像換皮路徑、Boss 像高血袋、塔升級是線性倍率、敘事多半關在圖鑑。
3. 擴充應走 **「加深度／可讀進程／可重玩差異」**，不是堆更多同質單位。

**成功標準（可驗收）：**

| 面向 | 現況痛點 | 擴充後應變成 |
|------|----------|--------------|
| 內容豐富度 | 單位多，但角色分工有重複 | 每塔／敵有「我為什麼建你／怕你」的一句话定位 |
| 故事性 | 世界觀在圖鑑完整，局中幾乎無感 | 波間／章節／Boss 有短劇情鉤子，動機在戰鬥中被提醒 |
| 耐玩度 | 重玩靠難度×地圖×詞綴，差異偏數值 | 每局有可講的「這局怎麼不一樣」故事與 build 分叉 |

**不做的事：**

- 不重寫核心戰鬥迴圈、不引入後端／複雜劇情引擎。
- 不為了劇情打亂 `generateWaveQueue` 決定性。
- 不一次做完整「章節戰役模式」再回頭補單位——**先快贏內容，再鎖深度系統**。

**出怪節奏（`rules.generateWaveQueue`）**

- 數量：`5 + floor(wave * 1.2)`；Boss 波減半再塞 1 隻 Boss。
- Boss：`wave % bossEvery === 0`；類型依 `floor(w/bossEvery) % 2` 在 `yaksha`／`boss` 間輪替。
- 主題偏壓：`waveTheme`（物理／雷／冰／火循環）+ `themeEnemyPool` 55% 抽偏。
- 解鎖式入場：盾／冰魄／熔岩／醫／武士／獵犬有最低波次門檻。
- **能力模板重用明顯**：`bloodrage`×3、`splitBat`×2、`shieldRegen`×2——機制池淺，外觀池深。

## 2. 敘事盤點：世界觀在，戰場上聽不到

### 2.1 已有的強項

`lore.js` 的骨架其實完整且有情緒：

- **世界觀三句話**：天樞崩裂 → 最後一界神火 → 英靈以魂晶締約守壇。
- **為何而守**：不是「擋怪拿分」，而是「神火若滅，山河魂魄歸於虛無」。
- **戰役編年 7 章**：覺醒之壇 → 裂界初潮 → 魔門啟封 → 王影墜壇 → 灰誓之軍 → 無星前線 → 最後神火；解鎖條件綁波次／Boss。
- **英雄列傳**：序承轉合四階，神魔誌角色（妲己、關羽、悟空等）完成度高。
- **UI 接點已在**：神魔誌 Codex、章節解鎖 toast、神諭低語。

### 2.2 薄弱處（玩家「在玩什麼故事」）

| 層級 | 現況 | 問題 |
|------|------|------|
| 開局動機 | 選地圖／難度就開打 | 沒有「這座聖壇今天為何危急」的 15 秒鉤子 |
| 局中節奏 | log：`第 N 波來襲`、事件標籤 | 事件有機械驚喜，**無敘事簽名** |
| 地圖 | 一句 desc | 三地像換路線的同一場仗，無文化地景 |
| Boss | 血厚＋輪替兩張臉 | 缺「這尊魔代表什麼威脅」的高潮感 |
| 女神 | HP 條＋升級 | 幾乎是資源槽，不是角色 |
| 戰役章 | 達成條件後進圖鑑 | **被動閱讀**，不是玩過一段劇情 |

**一句診斷：**  
敘事資產 = A；敘事投放 = C。玩家可以在 Codex 讀懂「為何守」，但在第 12 波未必感覺自己在守「最後神火」。

### 2.3 建議的敘事核心（擴充時統一口徑）

> **玩家是聖壇指揮官。**  
> 女神以魂晶召英靈；你以塔陣與策略替神火爭取時間。  
> 妖潮不是隨機刷怪，而是裂界「學會記住聖壇位置」的過程。  
> 每一波清空，是神火又亮了一寸；每一次漏怪，是記憶被啃掉一角。

後續所有 P0–P2 文案與章名應服務這條軸，避免另起一套西幻／科幻世界觀。

## 3. 「淺」處診斷（加內容深度的優先切口）

### 3.1 塔：角色清楚，成長無分叉

- 8 塔定位不錯（含 sniper／arcane 補單體與易傷）。
- 升到 10 級只是「同一招更痛更遠」→ **中後期 build 同質**。
- 全塔開局可用 → 無「解鎖帶來的進程爽感」。

### 3.2 敵：外皮多，機制模板少

- 能力 id 高度重用；中後期「又是加速／又是再生盾／又是分裂」辨識疲勞。
- Boss 兩種幾乎都是高 leak 高血包，**缺階段技、缺強制改陣型**。
- 醫官是唯一「必須優先擊殺」的戰術單位——同類應再有 1～2 個，但不要再複製醫官。

### 3.3 波次：變化有，章法弱

- 事件波是好系統，但玩家記憶點是「蟲潮／寶藏」而非「故事節點」。
- 無「第 5／10／15 波的儀式感」（Boss 有，章節敘事幾乎只在圖鑑）。
- 無盡與普通是同一生成器，**無盡缺里程碑敘事**。

### 3.4 地圖與詞綴：數值差異 > 體驗差異

- 地圖只改 path 與 goldMul；建塔思維差異有限（峽谷稍長）。
- 詞綴全是倍率；`aftershock` 是唯一「會講故事的機制」。
- 缺少「這張地圖專屬」的機關或敵偏壓。

### 3.5 進程與重玩

- 跨局進程：魂晶、英雄收集、羈絆、成就、編年——**偏 meta，不偏對局 build**。
- 局內進程：塔升等、女神、波次——線性。
- 重玩新意：難度×地圖×詞綴×英雄隊，但缺少「這局詞綴／章節改變了我的勝利條件」。

### 3.6 敘事投放缺口

| 應存在的投放點 | 現況 |
|----------------|------|
| 選圖時的地誌短文 | 一句 desc |
| 開局詞綴的神諭口吻 | 機械說明為主 |
| 事件波進場旁白 | 僅 label/desc |
| Boss 登場台詞 | 「Boss 出現」 |
| 章節達成局中演出 | toast + 圖鑑 |
| 地圖／塔／敵 Codex 深度 | 敵有 counterHint；塔無故事；地圖無 lore |

## 4. 擴充路線圖總覽

```text
P0 快贏（1～2 週可切片上線）
  新塔 2 + 新敵 3 + 事件波 2 + 關卡／波次簡述劇情 + 地圖短 lore
       ↓ 不改平衡骨架，只加可選內容與文案投放
P1 深度（系統分叉）
  Boss 階段波 + 塔升級分支 + 地圖機關 + Chapter 輕量戰役敘事
       ↓ 仍共用 generateWaveQueue 決定性
P2 延展（長線耐玩）
  無盡里程碑 + 難度詞綴疊加 + 元進程／成就擴編 + 收集圖鑑閉環
```

**平衡與決定性守則（全階段通用）：**

1. 新內容進 `config.js` 資料；生成邏輯變更走 `rules.js` 純函式，**可測**。
2. 任何隨機必須吃 `runSeed`／`affixSeed`／`wave` 混種 seed，禁止局中 `Math.random` 影響出怪。
3. 新塔／敵先用 emoji 或現有 PNG 變色／複用；不阻塞劇情與數值。
4. 預設波表成長率（`hpGrowthEarly/Late`）不為塞內容而陡升；新威脅用**機制**不是純加血。
5. 既有成就／任務 id 不改名；只追加。

## 5. P0 — 快贏：加料與「聽得見的故事」

**目標：** 一週內玩家感覺「單位變多、每局有旁白、事件更新鮮」，且不碰升級樹／章節模式。

### P0-1 新塔 ×2（補缺口，不搶舊塔飯碗）

#### A. `beacon`「引魂燈塔」

| 欄位 | 內容 |
|------|------|
| **預期體驗** | 支援位第二解：不增傷，改「暴露／緩速光環」——讓 sniper／arrow 好打高速與隱匿系。 |
| **機制草案** | 射程內敵人 `reveal`（若未來有隱匿）＋小幅減速 15%（低於 frost 的 50%）；可疊加規則：與 frost 取高不疊乘。`cost` 約 100～120。 |
| **敘事簽名** | 「魂晶點亮的路標——讓迷途者與妖影同時現形。」 |
| **實作要點** | `TOWERS.beacon`；`game` 光環 tick；顧問對 `fast` 波給分。 |
| **系統接點** | `config.TOWERS`、`game` 光環、`rules.recommendTowersForWave`／`towerReason`、`ui` 熱鍵 9、圖示可先 emoji 🏮。 |
| **平衡注意** | 不做傷害，避免再多一條 DPS 曲線；明確弱於 frost 的控場，強於「情報／輔助」。 |

#### B. `mortar`「墜星臼砲」（或命名「天劫砲」）

| 欄位 | 內容 |
|------|------|
| **預期體驗** | 超慢、超高爆、**最短射程外環**（不能打腳邊）——與 cannon 分職：cannon 清近群，mortar 炸中後段。 |
| **機制草案** | `fireRate` 低、`splash` 大、`minRange`；火元素。`cost` 約 160。 |
| **敘事簽名** | 「聖壇借用墜星殘骸——代價是無法守護自己腳下。」 |
| **實作要點** | 目標選擇：優先路徑進度中段；`minRange` 在 `acquireTarget` 過濾。 |
| **系統接點** | 同其他攻擊塔；投射物可複用 `cannonball`／`fireball`。 |
| **平衡注意** | 高傷害必須吃 minRange 與低射速；Boss 波有效但不應秒殺。 |

**刻意不在 P0 做的塔：** 召喚塔、佔格牆——會動到 pathing／佔用規則，風險高。

### P0-2 新敵 ×3（一個新機制族，兩個舊機制變體禁止）

#### A. `silencer`「緘口妖僧」— 新機制：`towerMute`

| 欄位 | 內容 |
|------|------|
| **預期體驗** | 靠近時讓最近 1 座塔「噤聲」數秒（不攻擊）；玩家要分散關鍵塔或優先擊殺。 |
| **數值感** | 中血中速；wave≥10 才進池。 |
| **敘事** | 「牠們不殺塔，只偷走塔的聲音——聖壇最怕的是寂靜。」 |
| **實作要點** | 能力在 `game` 敵人 tick：對距離最近塔設 `mutedUntil`（可與 aftershock stun 共用欄位語意）。 |
| **決定性** | 選塔規則固定（例如：距離最近 → 同距取建造序），禁止隨機。 |
| **系統接點** | `ENEMIES`、`pickDefaultEnemy`、`enemyAvailableInWave`、`inferLeakReason`、`counterHint`。 |

#### B. `mirrorling`「裂鏡童」— 新機制：`reflectOnce`

| 欄位 | 內容 |
|------|------|
| **預期體驗** | 每隻一生反射第一次「技能」傷害（或第一次高傷彈）；逼玩家用持續毒／多段 tesla，而不是一發隕石洩憤。 |
| **敘事** | 「鏡子裡的孩子只記得被雷劈的那一下。」 |
| **實作要點** | flag `reflectedSkill`；主動技能 `castSkill` 路徑判定。 |
| **系統接點** | 技能傷害入口統一標記 `source: "skill"`。 |

#### C. `warden`「裂界守門人」— 新機制：`auraArmor`

| 欄位 | 內容 |
|------|------|

### P0-3 事件波 ×2

| id | 名稱 | 規則草案 | 預期體驗 | 接點 |
|----|------|----------|----------|------|
| `eclipse` | 蝕火波 | 全場塔 `towerDamageMul` 0.85，敵 `reward`×1.4；視覺可僅 log／濾色 | 「神火被遮——輸出變弱但擊殺更肥」 | `EVENT_WAVES`、`createEnemy` 乘區、log 旁白 |
| `pilgrim` | 朝聖波 | 固定混入 1 隻「朝聖者」高 reward 低威脅 + 護衛群；擊殺朝聖者額外魂晶 0～1（若不想動魂晶則只給金） | 道德／效率選擇感（其實都該殺，但節奏像護駕） | queue 後處理 push 特殊 type；**seed 決定是否本事件**（已是事件表抽選） |

現有 5 事件 + 2 = 7；`getEventWave` 仍用 seed 索引，**長度變了會改變舊 seed 對應事件**——需在實作時二選一：

- **建議：** 新事件接在 keys 穩定排序後的固定表，並在 patch note 註明「事件表擴充會改變舊 seed 對應」；或  
- 用 `wave + runSeed` 映射到「加權表」並把舊 5 事件權重鎖住（較複雜，P1 再做也行）。

P0 務實做法：**接受事件表擴充改變對應**，但 **同一 build 的 queue 決定性測試仍要綠**。

### 現況一句話

> **神魔誌的殼與系統厚度已達「可長期營運的小型 PWA 塔防」；缺的是局中可感知的進程分叉與敘事投放，不是再從零寫世界觀。**

### 最值得做的三件事（若只能選三）

1. **波次／地圖／Boss 簡述劇情（P0-4）** — 成本最低、故事性立刻上升。  
2. **1～2 個新機制敵 + Boss 階段（P0-2 → P1-1）** — 直接加深度與話題。  
3. **塔升級分支（P1-2）** — 中期耐玩與 build 討論的主引擎。

### 刻意延後

- 複雜劇情演出、語音、多結局 AVG。  
- 永久戰鬥數值的元進度。  
- 完全手寫關卡波表（維護成本與決定性測試負擔高）。

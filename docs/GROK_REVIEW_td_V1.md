## 2. (1) 元素命中合成品質

### 2.1 接線與觸發面

| 事件 | 合成 profile | 觸發位置 | 備註 |
|------|--------------|----------|------|
| 毒霧命中 | `poison`：magic + smoke | `dealDamage` → `1441` | 非 mortar 路徑 |
| 冰命中 | `ice`：ice-ring + flash | `1442` | frost 等 |
| 雷命中 | `thunder`：spark + flash | `1443` | tesla **穿透每人**各噴一次 |
| 火命中（非 mortar） | `fire`：fire + flash | `1444` | **加農 splash 內每隻怪各噴一次**（因 `dealDamage` 仍走元素分支） |
| 臼砲 | `mortar`：flash + fire + smoke（中心一次） | `1407-1410` | `dealDamage` 內刻意跳過 mortar 元素層 — `1440` |
| 技能元素 | 同 `FX_PROFILES[sk.element]` | `1467` | 再疊 `burst(40)` + `ring` |
| 普通死亡 | `death`：flash + smoke | `1703-1704` | + burst/ring |
| Boss 死亡 | `boss`：flash/fire/smoke/magic | `1703-1707` | `criticalFx: true` |
| 物理系單體（弓／狙／奧術等） | **無 texture** | 僅 `burst(..., 5)` — `1423` | 戰場「元素對比」不完整 |

**判定**：元素分型**有接到對的 Kenney 素材**（毒＝magic、冰＝ice-ring、雷＝spark、火＝fire），這點 e2e 的 `textureKinds` 也只驗到「有層＋kind 字串」，**不驗觀感**。

### 2.2 時序參數表（核心）

來源：`FX_PROFILES`（`1567-1598`）+ `texturedImpact` life 抖動（`1626`）+ `drawParticle` 淡出／縮放（`2488-2502`）。

| profile | 層 | texture | size | life (s) | spin | blend | 出生有效 α 粗算¹ | 備註 |
|---------|----|---------|------|----------|------|-------|------------------|------|
| poison | 1 | magic | 58 | 0.46 | 1.8 | lighter | ≈0.83 | 主層尚可 |
| poison | 2 | smoke | 48 | 0.56 | rand | source-over | ≈0.66 | driftY −14 |
| ice | 1 | ice | 66 | 0.42 | −1.1 | lighter | ≈0.76 | 環形辨識最好 |
| ice | 2 | flash | 44 | 0.22 | rand | lighter | **≈0.40** | 關鍵閃光偏淡 |
| thunder | 1 | spark | 72 | 0.34 | 2.4 | lighter | ≈0.61 | 壽命最短主層 |
| thunder | 2 | flash | 38 | 0.18 | rand | lighter | **≈0.32** | 幾乎看不到「啪」 |
| fire | 1 | fire | 62 | 0.42 | rand | lighter | ≈0.76 | |
| fire | 2 | flash | 42 | 0.20 | rand | lighter | **≈0.36** | |
| mortar | 1 | flash | 92 | 0.28 | rand | lighter | ≈0.50 | 尺寸最大的「先閃」 |
| mortar | 2 | fire | 118 | 0.52 | rand | lighter | ≈0.90 | 主體 |
| mortar | 3 | smoke | 126 | 0.78 | rand | source-over | ≈0.66 | driftY −22；層次最佳 |
| death | 1–2 | flash/smoke | 50/60 | 0.22/0.52 | | | 0.40 / 0.66 | 普通死亡偏小 |
| boss | 1–4 | flash/fire/smoke/magic | 210… | 0.42–1.0 | magic 1.4 | | 0.66–0.90 | 宣傳可用的大招級 |

¹ 出生有效 α ≈ `min(1, life×2) × textureAlpha`；`textureAlpha` smoke=0.66、其餘預設 0.9（`1631`）。life 另有 `×(0.92+rand×0.16)` 抖動（`1626`）。

**共用曲線（所有 texture 粒子）**

| 參數 | 公式／值 | 視覺後果 |
|------|----------|----------|
| 全域 fade | `a = clamp(life × 2, 0, 1)` — `2489` | life&lt;0.5 的層**出生就半透明**；短命 flash 幾乎沒有「第 0 幀全白」 |
| 尺寸曲線 | `scale = 0.72 + (1−ratio)×0.42` → **0.72→1.14** — `2495` | **由小放大再消**＝煙霧消散語意，不是 hit-stop／punch-in |
| 旋轉 | `rotation += spin × fxDt` — `1049` | 元素有差：冰逆轉、雷快轉、毒慢轉 |
| 重力 | texture **不受重力** — `1050` | 命中層黏在點上微飄（smoke 用 driftY） |
| low 檔 | 只留 `profile[0]`，size×0.78 — `1622-1625` | 元素只剩「一片染色主層」 |
| reduced | 整段 `texturedImpact` return 0 — `1619` | 正確；宣傳請關 reduced |

### 2.3 tint 合成品質

```text
tintedFxSprite: draw 原圖 → source-in 實心 color 填滿 → cache
（game.js:1600-1616）
```

| 優點 | 代價 |
|------|------|
| 離屏 cache、同色重用便宜 | **灰階細節被抹成單色 silhouette**；Kenney 火／煙的明暗層次幾乎只剩 alpha 邊緣 |
| 元素色可對齊塔色 | 冰 `#7dd3fc`、雷 `#fde047`、火塔色、毒 `#4ade80` 對了，但「材質感」弱 |
| 192×192 中間解析 | 放大到 100px+ 的 mortar／boss 層會糊，靠 lighter 混色遮醜 |

**合成節奏**：各層**同幀同座標生成**，靠 **life 長短** 假裝時序（flash 先死 → 主體 → smoke）。這是正確的低成本技巧；臼砲 0.28／0.52／0.78 的落差最好。元素命中只有 2 層且 flash life 過短＋出生 α 過低，**「先亮一下」幾乎不成立**。

### 2.4 高頻 thrash 與不一致

| 情境 | 行為 | 風險 |
|------|------|------|
| 寒冰 1.45 發/s | 每發 2 texture 層 | 多塔時 decor 粒子海；cap 220 擠舊（`1682-1684`） |
| 電磁 pierce 3 | 最多 3 目標 × 2 層 | 一發可 6 texture + 無額外 ring |
| 加農 splash N | **每隻** `dealDamage` 噴 fire 2 層 | 群傷畫面糊成橘霧；與臼砲「中心一次」不一致 |
| 物理塔 | 無 texture | 混編防禦截圖時，只有冰雷毒火「有質感」 |

### 2.5 小結 — 元素命中

| 面向 | 評等 | 說明 |
|------|------|------|
| 元素可辨（冰環／雷花／毒霧／火） | **PASS～RISK** | 靜止看 profile 可分；動態短命＋淡 α 後差距縮 |
| 打擊「厚度」（flash→body→smoke） | **RISK** | 只有 mortar／boss 有；元素命中 flash 太弱 |
| 材質／光線品質 | **RISK** | solid tint 砍掉 Kenney 灰度 |
| 接線正確性 | **PASS** | 毒／冰／雷／火／技能／死亡／Boss 有；物理缺席是取捨 |
| 測試 | **NOTE** | e2e 驗「有層／有 cache」，不驗 alpha、時序、觀感 |

**總評 (1)**：**RISK / P1** — 工程接線合格，**合成時序與淡出曲線還沒到「截圖級元素爆炸」**。

## 3. (2) 塔升級外觀 — Lv1–10 真的一眼可辨嗎？

### 3.1 參數公式（`towerVisualStyle` + `drawTower`）

| 量 | 公式 | 來源 |
|----|------|------|
| progress | `(lv−1)/(max−1)`，max=10 | `2261-2264` |
| tier | lv≤1 → 0；否則 `1+floor(progress×4)` clamp 到 0–4 | `2265-2266` |
| auraColor | `["#64748b","#38bdf8","#a78bfa","#facc15","#fb7185"][tier]` | 同上 |
| baseR | `CELL×(0.41 + progress×0.11)` | `2269` |
| spriteSize | `CELL×(0.70 + progress×0.18)` | `2270` |
| ringCount | `1 + floor(progress×3)`；**僅 lv≥2 才畫環** | `2271`、`2284` |
| gemSize | `3.5 + progress×6.5` | `2272` |
| 底座填色 | tier0 深灰半透；否則 aura + `38`/`52` hex α | `2295` |
| 寶石 | lv1 銀灰；之後 aura；tier≥2 白框 | `2307-2311` |
| LV 徽記 | **僅 lv&gt;1** 畫 `LV n` | `2329-2333` |
| 滿級光點 | **僅 lv≥max** 四點軌道 | `2315-2321` |
| 呼吸 pulse | `sin` 振幅 `0.012+progress×0.025` | `2280` |

### 3.2 「一眼可辨」判定

| 距離／條件 | 可辨粒度 | 判定 |
|------------|----------|------|
| 讀 `LV n` 字（10px bold + 描邊） | 精確 1–10 | **PASS（靠文字）** |
| 遠看只看色 | **5 檔**：灰／藍／紫／金／粉 | **PASS（粗分）** |
| 遠看只看環數 | **4 檔**：0／1／2／3／4（Lv10） | **PASS（粗分）** |
| 無字、相鄰級（2↔3、4↔5、8↔9） | baseR 差 &lt;1px、sprite 差 &lt;1px、同色同環 | **FAIL 一眼** |
| 剪影／造型語言 | 同一張 `assets/towers/{type}.png`，只 scale 0.70→0.88 CELL | **無造型階梯** |
| 滿級 | 軌道光點 + 第 4 環 | **PASS（終點有獎）** |

**對 Codex「清楚級差」的校正**：

- **清楚的是「段位」（約每 2 級一色）+ 數字**，不是「Lv1 到 Lv10 每級一個獨立剪影」。
- e2e 只比 `lvMax.gemSize > lv1`、`ringCount`、`tier`（`test-td-e2e.js` R57 guard）——**端點成長有、中段階梯無**。

### 3.3 其他視覺副作用

| 點 | 說明 | 等級 |
|----|------|------|
| 多塔同場 | 高 tier `shadowBlur = 7+tier×4`（`2286`）易光暈糊成一片 | P2 |
| 色與元素撞車 | 金＝雷／聖光、藍＝冰、粉＝引魂燈；**升級色階不是元素語意**，混讀成本 | NOTE |
| pulse 振幅 | 滿級仍只有 ~3.7% scale，**截圖幾乎靜止**，「呼吸」賣點弱 | P2 |
| 支援塔 | 同樣走 `drawTower`，聖光／引魂也會長粉金光環，可能誤讀成「攻擊塔很強」 | NOTE |

**總評 (2)**：**RISK / P1** — **不能誠實宣稱「Lv1–10 一眼可辨」**；應改寫為「五段色階 + LV 字 + 滿級特效」。

## 4. 三圖氛圍／呼吸光（截圖語境附帶）

| 地圖 | ground | tint α | breath | 路紋 | 一眼記憶 |
|------|--------|--------|--------|------|----------|
| plains | `#0e1a14` | **0.035** 綠 | 薄荷 0.12 | footprints α≈0.22 | **最弱**；近乎預設草地 |
| canyon | `#221912` | **0.24** 暖赭 | 琥珀 0.11 | slabs | 中上 |
| lava | `#1d1014` | **0.32** 紅褐 | 玫瑰 0.12 | cracks | 最強 |

- 氛圍層畫在 path **之後**（`1789-1790`），tint 會蓋路——正確。
- 路紋預烘焙 cache（`2074-2109`），對比刻意低；**宣傳縮圖後路紋易消失**。
- 呼吸光：`strength = 0.48 + (sin(t×1.25)+1)×0.16`，中心 α≈`0.16×strength`（約 0.08–0.13）— `2064-2070`。  
  **條件：波間 + 非 reduced + 非 low**。戰鬥中、auto-low 手機、reduced 皆無。

**總評**：功能與 e2e「三 theme 字串互異」**PASS**；**宣傳用戰鬥截圖**下平原與呼吸光貢獻接近 0 → **RISK / P2**。

## 5. (3) 宣傳截圖標準 — 3 個最划算下一步

評選標準（本報告採用）：

1. **截圖 0.5 秒內**能讀出：這是塔防、有元素、塔有成長、地圖有個性。  
2. **工時／風險比**：少改數值、少動 RNG、少重繪全套立繪。  
3. **現況缺口最大且參數已有掛點**（避免從零發明系統）。

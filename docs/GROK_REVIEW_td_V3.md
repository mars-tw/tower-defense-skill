## 2. (1a) 升級色錨解耦 — 還會不會「彩虹搶元素」？

### 2.1 V2 問題回顧

R58 修好「級」的 10 色階梯後，**藍／青／紫／金／粉**同時驅動環、shadow、底座填色、寶石、LV → 混編終盤像霓虹塔林，**讀不出塔種元素**。

### 2.2 R59 雙通道（現況）

| 通道 | 驅動色 | 用途 |
|------|--------|------|
| **元素錨** | `def.color`（塔種表） | 主陰影、內環、底座多邊形填／描邊、（非金屬）承台描邊 |
| **升級刻度** | `auraColor`（`levelColors[lv-1]`） | **僅**最外環、頂部寶石、LV 字、寶石 shadow |
| **質量殼** | 金屬灰（固定 rgba） | `metal` 塔的深色承台填色＋亮邊＋鉚釘 |

關鍵行：

```text
// game.js:2347-2376（摘要）
shadowColor = def.color; shadowBlur = glow.baseBlur
承台 fill/stroke = metal 灰 或 def.color
rings: i === 最外 ? auraColor : def.color
底座 fill/stroke = def.color（α 隨 tier）
// 寶石 / LV 仍 auraColor（2385-2412）
```

### 2.3 塔種元素色（錨點是否合理）

| 塔 | element | `def.color` | 混編可讀性 |
|----|---------|-------------|------------|
| arrow | physical | `#a3a3a3` | 灰錨 OK |
| sniper | physical | `#94a3b8` | 灰藍錨 OK |
| cannon | fire | `#f97316` | 火橘 OK |
| mortar | fire | `#fb923c` | 淺火、與加農可分 |
| frost | ice | `#38bdf8` | 冰藍 OK |
| tesla | thunder | `#facc15` | 雷金 OK |
| poison | physical* | `#22c55e` | 毒綠（語意色，非元素表）OK |
| arcane | physical | `#a855f7` | 奧術紫 OK |
| support | physical | `#fde047` | 聖光金 OK |
| beacon | physical | `#fb7185` | 粉紅錨 OK |

\* `poison.element === "physical"` 但色相綠；承台走 `profiles.poison` 金屬殼，不靠 element 表。

### 2.4 裁決

| 問題 | R58 | R59 | 裁決 |
|------|-----|-----|------|
| 全環彩虹搶元素 | 是 | **內環元素、外環升級** | **PASS** |
| 底座填色 = 升級色 | 是（tier 六角 alpha） | **改 `def.color` α** | **PASS** |
| shadow 色 = 升級色 | 是 | **`def.color`** | **PASS** |
| 寶石／LV 仍彩虹 | — | **刻意保留** | **PASS（設計）／NOTE（高 Lv 仍有色點）** |
| 殘餘彩虹面積 | 大 | 縮到最外環＋寶石＋字 | **PASS（主訴關）**；Lv9–10 五環時最外仍醒目屬預期刻度 |

**總評 (1a)**：**PASS** — V2-A 主訴關閉。誠實寫法：可宣稱「塔種色錨主導、升級彩虹降為外階刻度」；不宜宣稱「畫面上完全沒有升級色相」。

## 3. (1b) 光暈固定預算 — thrash 有沒有消？

### 3.1 政策表（`towerGlowPolicy` `2297-2312`）

| 條件 | budget | baseBlur | gemBlur |
|------|--------|----------|---------|
| low 或 reduced | 0 | 0 | 0 |
| count ＜ 10 | 8 | 選取 8／maxed 7／一般 5 | 10／9／7 |
| 10 ≤ count ＜ 16 | 6 | 同上；若 count＞12 則 dense 3／4 | dense 壓低 |
| count ≥ 16 | 4 | dense 3（選取仍 8） | dense 4（選取 10） |
| 未入選 | — | **0** | **0** |

分配規則：`enabled = count <= budget || index < budget || selected`  
→ **固定繪製索引**，不依 frame 輪替；選取塔可 **超出 budget 仍亮**。

### 3.2 重算（與 e2e 對齊）

| 場景 | 結果 |
|------|------|
| 18 塔、皆未選、Lv8 | enabled **恰好 4**（index 0–3）；其餘 base/gem blur=0 |
| index=17 且 selected | enabled，baseBlur=8、gemBlur=10 |
| low／reduced 即使 selected | enabled=false，blur=0 |
| budget 階 | count 4→8、9→8、10→6、15→6、16→4 |

e2e：`denseGlow.filter(enabled).length === 4` 且 slice(4) 全零；low／reduced 零（`1324-1327`）。

### 3.3 與 R58 對照

| 項 | R58 | R59 |
|----|-----|-----|
| 每塔 base blur | `5 + tier×3`（最高 ~20）全開 | 預算內 3–8，預算外 **0** |
| gem blur | `5 + progress×13`（~5–18）全開 | 預算內 4–10，預算外 **0** |
| 幀間閃爍 | 全開時 thrash 來自「全亮」而非輪替 | **無輪替**；穩定少光暈 |

### 3.4 殘餘 NOTE

| ID | 說明 | 優先 |
|----|------|------|
| V3-G1 | 名額綁 **陣列前 N 座**（通常早蓋的塔），後蓋高 Lv 塔預設無光暈——**穩定但不公平**；選取可補，未選則終盤「光暈集中在前排」 | P2 |
| V3-G2 | e2e 只驗 policy 回傳值，**不驗實際 `ctx.shadowBlur` 像素** | NOTE |
| V3-G3 | dense 門檻（count＞12）與 budget 門檻（≥10／≥16）是兩套刻度，合理但需讀 code 才懂 | NOTE |

**總評 (1b)**：**PASS** — V2-B「後期糊板 thrash」主訴關閉。剩餘是分配公平性（P2），不是「還在逐幀亂閃」。

## 4. (1c) 物理厚度／加農重量 — 補的是哪一層？

### 4.1 材質表（`towerMaterialStyle` `2314-2325`）

| type / fallback | mass | rim | rivets | metal |
|-----------------|------|-----|--------|-------|
| arrow | 0.98 | 2.6 | 4 | true |
| sniper | 1.02 | 3.0 | 4 | true |
| cannon | **1.08** | **3.4** | 4 | true |
| mortar | **1.16** | **4.4** | **6** | true |
| poison | 0.96 | 2.4 | 3 | true |
| element==physical 其餘 | 0.94 | 2.2 | 3 | true |
| 非金屬（frost／tesla／arcane 等） | 0.91 | 1.8 | 0 | false |

單調性（e2e 鎖）：`arrow.mass < cannon.mass < mortar.mass` 且 `mortar.rim > cannon.rim` — **PASS**。

### 4.2 繪製落點（`2350-2361`）

- 金屬：深 slate 填（`rgba(15,23,42,.88)`）＋亮邊 stroke ＋ 圓周鉚釘點  
- 非金屬：較淡底＋`def.color` 描邊、無鉚釘  
- 承台半徑 = `baseR * mass`（加農／臼砲明顯「坐得更寬」）

### 4.3 與 V2-C 訴求的對齊度（重要）

V2 下一步 C 原文包含兩截：

1. **塔體乾** → 金屬承台／重量階梯 → **R59 已做（PASS）**  
2. **命中乾**（物理 `burst(...,5)` 無 texture；加農 fire 雙層薄於臼砲三層）→ **R59 未動**

現況命中路徑（仍同 R58）：

| 塔／事件 | 命中視覺 | 位置 |
|----------|----------|------|
| 冰／雷／毒／火（非 mortar／cannon splash） | `texturedImpact` 2 層 | `1445-1448` |
| cannon splash | 中心一次 `fire` 雙層（flash+fire） | `1407-1408` |
| mortar splash | 中心 `mortar` 三層 + burst28 + ring + shake | `1410-1414` |
| 物理單體（箭／狙／奧術等） | **僅** `burst(..., 5)` | `1426` |

**總評 (1c)**：**PASS（塔體質量）／NOTE（命中厚度未進 R59）** — Codex 宣稱範圍寫的是「物理／重砲厚度」與 mass/rim 數字，**就宣稱本身為真**；不要誤讀成「物理命中也有 Kenney 層了」。

## 6. 剩餘最大缺口（收官後）

### 🥇 最大缺口 — 命中反饋「元素濕／物理乾」＋加農仍薄於臼砲一截

| 項目 | 內容 |
|------|------|
| 為何仍是最大 | R59 把「塔站在那裡」變厚了，但 **money shot 的動感在開火／命中瞬間**。混編火力秀時：冰雷毒火有 Kenney 層，弓／狙／奧術只有 5 點 burst；加農有爆心但不含 smoke／ring，臼砲仍是唯一「大炸」 |
| 證據 | 物理 `1426` 無 `texturedImpact`；加農 `1407-1408` 用 `FX_PROFILES.fire`（2 層）；臼砲 `1410-1414` + `1572-1575`（3 層＋ring＋shake） |
| 建議方向（不實作） | (1) 物理短命 **flash 或 spark 單層**（low 仍 1 層邏輯可共用）；(2) 加農可選 **小 smoke** 或精簡第三層，**維持中心一次**；(3) 不要改回 splash 每怪 texture |
| 預估成本 | 0.5–1 人日 |
| 優先級 | **P1**（宣傳混戰對比）；非 P0（可玩性／正確性無損） |
